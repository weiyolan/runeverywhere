## Goal

Ship a freemium "RE Pro" auto-renewing subscription (monthly + annual) using RevenueCat as the billing/entitlement layer, pre-launch, so v1 submits with the paywall live. Pro gates exactly three feature sets: **integrations** (Strava + Garmin connect; HealthKit stays free), **advanced stats** (full run history beyond the 10 most recent + GPX export), and **profile flair** (PRO chip, accent color, avatar ring). The core social loop (create/discover/join/chat/record/points/reviews) and **every safety feature (SOS, live-share, trusted contacts) stay free forever** ŌĆö paywalling safety is an App Review and ethics problem. Client entitlement truth is the RevenueCat SDK; the server mirrors it to `profiles.pro_until` via webhook so Edge Functions and RPCs enforce gates server-side (PLAN.md ┬¦2 tiers). A `monetization` feature flag fails **open** (everyone Pro) so a store emergency can never brick features.

### C ŌĆö Server mirror: migration `00000000000065_pro_entitlement.sql` + webhook

**C1. Columns + guard.** `alter table public.profiles add column pro_until timestamptz, add column flair_accent text check (flair_accent in ('volt','discover','challenge','social')), add column flair_ring boolean not null default false;` Column grants per the P0/P1 pattern: `grant update (flair_accent, flair_ring) on public.profiles to authenticated;` ŌĆö `pro_until` is deliberately **not** granted (server-cache rule, PLAN.md ┬¦2). Guard trigger `profiles_flair_requires_pro` (BEFORE UPDATE): if either flair column changes and `coalesce(new.pro_until, 'epoch') <= now()` ŌåÆ `raise exception 'pro required'`.

**C2. Helper.** `create function public.is_pro (p_user uuid) returns boolean language sql stable security definer set search_path = '' as $$ select coalesce((select pro_until from public.profiles where id = p_user), 'epoch'::timestamptz) > now() $$;`

**C3. Events table.** `create table public.revenuecat_events (id text primary key, type text not null, app_user_id uuid, environment text not null, payload jsonb not null, received_at timestamptz not null default now());` RLS enabled, **no policies** (service-role only), plus the P6 A2-style revoke so client access fails loudly.

**C4. Edge Function `supabase/functions/revenuecat-webhook/index.ts`.** `verify_jwt = false` (`[functions.revenuecat-webhook]` in config.toml). Steps: (1) require `Authorization === Bearer ${REVENUECAT_WEBHOOK_SECRET}` (function secret; also configured in RC's webhook settings) else 401; (2) `insert ŌĆ” on conflict (id) do nothing` into `revenuecat_events`; if no row inserted ŌåÆ 200 (duplicate); (3) for types `INITIAL_PURCHASE|RENEWAL|UNCANCELLATION|PRODUCT_CHANGE|EXPIRATION|BILLING_ISSUE` with entitlement `pro`: `update profiles set pro_until = to_timestamp(event.expiration_at_ms / 1000.0) where id = event.app_user_id` (service role client); `CANCELLATION` = no-op (entitlement runs to period end); `TRANSFER`: null out `pro_until` for `transferred_from`, recompute for `transferred_to` from the event; unknown types ŌåÆ log + 200. Anonymous RC ids (`$RCAnonymousID:ŌĆ”`) don't cast to uuid ŌåÆ skip with a log line. Deploy + `supabase secrets set REVENUECAT_WEBHOOK_SECRET=ŌĆ”`. Acceptance: RC dashboard "Send test event" ŌåÆ row in `revenuecat_events`, `pro_until` updated.

**C5. History limit.** `create or replace function public.list_past_runs()` (same signature/body as P4 D5) with a final `limit case when public.is_pro((select auth.uid())) then null else 10 end` (implemented via a wrapping subquery ŌĆö SQL `limit null` = no limit). **C6. `delete-account` extension.** Add to P5 F8's function, before returning: `await fetch('https://api.revenuecat.com/v1/subscribers/' + uid, { method: 'DELETE', headers: { Authorization: 'Bearer ' + REVENUECAT_SECRET_KEY } })` (new function secret; tolerate 404). ## Risks & gotchas

| Risk | Mitigation |
|---|---|
| Apple 3.1.2 rejection (subscription disclosures) | Paywall carries price/term/auto-renew text + ToS/Privacy links + RESTORE; reviewer note (G3) points at them; P7 attaches products to the first submission |

## Decisions made by this plan

1. **Model: subscription-only** ŌĆö monthly $4.99 + annual $29.99 (~50% saving), one entitlement `pro`, offering `default`. Prices are store-console config, not code; no lifetime product, no intro/trial offers in v1 (smaller review surface; add as a post-launch experiment).
2. **Paywall boundary**: integrations (Strava/Garmin; HealthKit free) + advanced stats (history >10 runs, GPX export) + flair. **Host limits rejected** ŌĆö the core loop stays fully free; **safety is never paywalled**.
3. **Pre-launch ship**: P6.5 executes between P6 and P7; P7 shifts to weeks 22ŌĆō24 and submits v1 with the subscription attached.
4. **Custom paywall screen**, not `react-native-purchases-ui` remote paywalls ŌĆö design-system fidelity (Saira/Volt) outweighs remote-config iteration for v1.
5. **Identity**: RC app user id == Supabase uid via `logIn`/`logOut`; anonymous boot config; webhook and deletion key on the uid.
6. **Server mirror = `profiles.pro_until`** (single timestamp), not an entitlements table ŌĆö one entitlement exists; `revenuecat_events` keeps the raw audit trail and idempotency key.
7. **Fail-open kill switch**: `feature_flags.monetization` OFF ŌćÆ everyone Pro, paywall hidden ŌĆö a billing outage or review emergency can only ever *give* features away.
8. **Expiry semantics**: `CANCELLATION` keeps Pro until period end; lapse blocks *new* connects/exports/history-depth but existing `connected_accounts` keep importing (punishing lapsed users by silently breaking sync invites refunds/reviews); flair is render-time gated so no cleanup job.
9. **History cap enforced server-side** in `list_past_runs` (free = 10 newest); GPX export enforced in the `export-gpx` Edge Function; flair enforced by trigger ŌĆö every gate has a server layer; client checks are UX only.
10. **GPX built from raw samples** (P4's gzipped Storage objects) in an Edge Function ŌĆö polylines lack timestamps/elevation; export is the first consumer of the raw-sample archive.
11. **Migration slots `ŌĆ”65`ŌĆō`ŌĆ”69`** borrowed from P6's range so filename order matches execution order without renumbering P7 (its docs and cross-references stay intact).
12. **Sandbox events accepted** on hosted during this phase (it *is* the test bed pre-launch); `RC_IGNORE_SANDBOX` env flips after launch.
